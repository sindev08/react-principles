TanStack Query Pattern

A declarative pattern for managing asynchronous state and server synchronization. It abstracts away the complexities of caching, background updates, and stale data.

01

Principle

The core principle behind TanStack Query is treating server state differently than client state. Most "state" in modern applications is actually a cache of server data. Instead of manually using useEffect and useState, you describe the resource you need via a unique key, and the pattern handles the lifecycle, caching, and synchronization automatically.

lightbulb

Pro-tip: This pattern effectively removes the need for global state management for most data-driven UI components.

02

Rules

  • check_circle
    Unique Query Keys Always use array-based keys that reflect all dependencies of the fetcher.
  • check_circle
    Custom Hook Encapsulation Wrap your query logic in a custom hook for reusability and cleaner UI components.
  • check_circle
    Error Boundaries Leverage built-in error handling to provide consistent feedback to users.
  • check_circle
    Stale-While-Revalidate Configure stale times to prevent unnecessary network requests for stable data.
03

Pattern

abstract-pattern.ts 

import { useQuery } from '@tanstack/react-query';

// Abstract custom hook pattern
export const useResource = (id: string) => {
  return useQuery({
    queryKey: ['resource', id],
    queryFn: () => fetchResource(id),
    staleTime: 1000 * 60 * 5, // 5 mins
  });
};
04

Implementation

info

Version Compatibility

Requires TanStack Query v5+ and React 18+. For older versions, some API syntax may vary (e.g., query arguments).

In Next.js, the pattern emphasizes hydration. You fetch data on the server, dehydrate the state, and hydrate it on the client to ensure immediate availability and SEO benefits.

app/posts/[id]/page.tsx 

import { HydrationBoundary, dehydrate } from '@tanstack/react-query';
import { getQueryClient } from './get-query-client';

export default async function PostPage({ params }: { params: { id: string } }) {
  const queryClient = getQueryClient();

  await queryClient.prefetchQuery({
    queryKey: ['posts', params.id],
    queryFn: () => getPost(params.id),
  });

  return (
    // Hydrate the client side cache
    <HydrationBoundary state={dehydrate(queryClient)}>
      <PostDetails id={params.id} />
    </HydrationBoundary>
  );
}
