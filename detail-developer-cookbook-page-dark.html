TanStack Query: Standardized Data Fetching

v5.0.0

Last updated: Oct 24, 2023 • 8 min read

1. Principle

The core principle of the TanStack Query Pattern is the strict separation of Server State from Client State. Unlike traditional state managers (Redux, Context), TanStack Query treats fetched data as a cache that requires synchronization, rather than a static global variable.

  • check_circle Declarative data fetching over imperative effects.
  • check_circle Zero-config caching and background revalidation.
  • check_circle Shared async state without global providers.
  • check_circle Standardized loading and error boundaries.

2. Rules

cancel

Don't use useEffect for fetching

Avoid managing loading/error states manually with useState inside a useEffect hook. It leads to race conditions and "zombie children" states.

check_circle

Do wrap fetches in Custom Hooks

Always export query logic as reusable custom hooks (e.g., `useUserData`). This encapsulates the query key and fetch function.

info

Use Stable Query Keys

Query keys should be serializable and follow a consistent structure: `['scope', id, { filter }]`.

3. Pattern Breakdown

database

Query Key

The unique identifier that maps to your data in the cache.

bolt

Fetch Function

A promise-returning function that retrieves your data.

cached

Stale Time

Defines how long the data is considered fresh before re-fetching.

account_tree

Hydration Architecture

TanStack Query facilitates a seamless handoff between Server Components and Client components using the <HydrationBoundary /> pattern.

4. Implementation

hooks/useProfile.ts 
import { useQuery } from '@tanstack/react-query';

// The Fetch Function
const fetchUser = async (userId: string) => {
  const response = await fetch(`/api/users/${userId}`);
  if (!response.ok) throw new Error('Failed to fetch user');
  return response.json();
};

// Custom Pattern Hook
export const useUserProfile = (userId: string) => {
  return useQuery({
    queryKey: ['user', userId],
    queryFn: () => fetchUser(userId),
    staleTime: 1000 * 60 * 5, // 5 minutes
    retry: 2,
  });
};

tips_and_updates Expert Tip

Combine TanStack Query with Zod for runtime type validation. Wrapping your fetch function response in a schema parse ensures your client state is always valid and strictly typed.
